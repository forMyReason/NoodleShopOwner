故事是这样的。

前两天我在 Cursor 里跟 Agent 聊财报，随口丢了一句，帮我对比一下苹果和腾讯。

然后它回了我一堆 PE、PB、ROE，表格整整齐齐，结论也整整齐齐，什么估值合理、建议关注、值得关注。

我盯着屏幕看了十秒。

= =

我知道它没骗我，数字大概率是对的。但我就是没感觉。

苹果 PB 四十倍，腾讯 PB 三点九倍，差了十倍。然后呢？我脑子里还是空的。一个是三万多亿美元，一个是四万多亿港币，单位不一样，体量不一样，你拿两个裸倍数横比，就像拿两家面馆一个在北京三里屯、一个在贵阳老街，只比「租金是收入的多少倍」，完全不知道盘下来是什么体感。

我就琢磨，有没有一种办法，把这件事讲到我这种普通人能秒懂的程度。

后来我真的把它做成了一个 Skill，叫 **noodles-shop-owner**，GitHub 仓库名 **NoodleShopOwner**。

核心就一句话，把所有公司的市值，统一缩放到 **100 块**，然后用一个**极具犀利商业眼光的面馆老板**的视角，读财报。

---

坦率的讲，这个方法不是我发明的什么高深公式。

它叫 **100 块法**，也有人叫市值锚定法、估值归一化透视法，名字听着唬人，逻辑简单到你想笑。

不管这家公司市值是几万亿还是几百亿，一律当成老板开价 **100 块** 的面馆。然后只算四件事。

第一，盘店开价，100 块，锚死。

第二，账面家底，100 除以 PB。PB 四十，家底就是 2.5 块。PB 四，家底就是 25 块。

第三，过去一年净赚还是净亏。盈利公司，100 除以 PE。亏损公司，不走 PE 除法那条路，用净利润除以实际总市值再乘 100，结果是负的，该负就负。

第四，家底打工效率，利润除以家底，就是 100 块口径下的 ROE。

就这四步。

你别看它简单，苹果和腾讯放一起，体感是完全不一样的。

---

我举个真实的例子，数字来自 Skill 里自带的演示，苹果 vs 腾讯 2025 年财报，**不是实时行情，别拿去下单**。

苹果这边，PE 大概 35 倍，PB 大概 40 倍。

换成面馆语言，你花 100 块盘下这家店，账面家底只有 **2.48 块**，过去一年净利润 **2.84 块**，100 块口径 ROE **114.5%**。

你品品这个数。

家底两块四，一年赚两块八。

ROE 破百。

这说明什么，说明苹果几乎不是靠厂房设备存货在赚钱，靠的是品牌、iOS 生态、App Store、芯片设计、用户粘性，这些东西在会计账本上几乎不算「家底」，但每年哗哗进账。

所以 PB 四十倍看起来贵到离谱，但 PE 三十五倍、ROE 一百多，你花 100 块买到的是一个每年能吐 2.84 块的印钞机，不是一堆锅碗瓢盆。

腾讯呢，PE 大概 21 倍，PB 大概 3.9 倍。

同样 100 块盘店，账面家底 **25.64 块**，过去一年净利润 **4.76 块**，ROE **18.6%**。

家底厚了十倍不止，赚钱也更多，4.76 块 vs 2.84 块。但 ROE 只有苹果的六分之一左右。

这就是两种完全不同的店。

苹果是几乎无家底，但赚钱能力炸裂，靠品牌和生态活着。

腾讯是家底扎实，赚钱稳健，像一个经营了多年的老店，锅碗瓢盆都在，投资资产、固定资产、现金储备都看得见。

放到终极对比表里，一行苹果，一行腾讯，100 块标尺统一，你不用再脑内换算三万亿和四万亿港币，一眼就能看出，贵的贵在哪里，便宜的便宜有没有毒。

---

说到这个，我觉得这个方法真正值钱的，不是公式本身。

公式谁都能写。

值钱的是 **Agent 怎么讲**。

我在这套 Skill 里给 Agent 立了一个角色，不是分析师，是**准备掏 100 块盘店的人**。

它必须做几件事。

把数字翻译成面馆语言，不能复读 PE 合理建议关注那种 AI 八股。

每个公司至少给一条反直觉洞察，比如苹果 ROE 破百到底反常在哪里，腾讯 Non-IFRS 利润和归母利润差 348 亿又说明什么。

「你的评价」那一列，必须是判断句，不能是「PE 21 倍 PB 3.9 倍」复读机。

公式旁边要括号展示计算过程，2.84 块，括号里写 100 除以 35.22，让你能验。

还有一条我很在意的，**停损**。

关键字段缺失的时候，Agent 只能报已经采到的数据，列缺失清单，请用户补充，**禁止**硬下终极结论，禁止编造数字。

我知道很多 AI 分析财报的翻车，不是算错了，是缺数了还装懂。这套 Skill 里我直接把这条路堵死了。

亏损公司也有专门路径，PE 小于等于零的时候，**禁止** 100 除以 PE，利润走市值缩放，对比表里 PE 那列填「无效（亏损）」，不能填负十五倍那种鬼东西。

我还放了一个 Python 验算脚本 calc_anchor.py，纯标准库，没第三方依赖，盈利路径亏损路径都能跑，--self-check 一键自检。

你信不过 Agent 口算，自己敲一行命令验。

---

可能有小伙伴纳闷，这不就是 PE PB ROE 换了个马甲吗。

是，也不是。

换马甲的部分，是公式确实就那几个，没凭空造指标。

不是换马甲的部分，是你终于能 **横向体感** 了。

以前苹果 PB 四十、腾讯 PB 三点九，你知道差十倍，但没有直觉。

现在 100 块口径下，苹果家底 2.48 块，腾讯家底 25.64 块，你立刻感觉到，一个是在买印钞机，一个是在买带家底的老店。

多公司对比的时候，Skill 会出一张「N 家面馆排排坐」的终极对比表，再加核心洞察和老板一句话。

老板一句话不是投资建议，是盘完店之后的那种判断，值不值得盯，贵有没有毒，便宜是不是家底有毒。

再说一遍，**这是估值透视和财报解读工具，不是投资建议**。数据以你指定的报告期和市场来源为准，Agent 联网搜，搜不到就停，不编。

安装也简单，我尽量没搞复杂。

GitHub 搜 forMyReason/NoodleShopOwner，克隆下来，把内层 noodles-shop-owner/ 文件夹复制到 Cursor 的 skills 目录就行。

项目级放 .cursor/skills/noodles-shop-owner/，全局放 ~/.cursor/skills/noodles-shop-owner/。

Windows PowerShell 一行的事。

git clone https://github.com/forMyReason/NoodleShopOwner.git
cd NoodleShopOwner
cp -r noodles-shop-owner ~/.cursor/skills/noodles-shop-owner

支持 Cursor、Codex、Claude Code。

用起来更懒，Agent 对话里 @noodles-shop-owner，或者说「100 块法」「对比苹果和腾讯」「哪家更值得盘」，它就走完整 SOP，搜数据、算四步、输出面馆语言、自检、给你老板一句话。

完整输出长什么样，仓库里 examples.md 有苹果 vs 腾讯的双公司演示，公式边界和亏损路径在 reference.md，Agent 行为定义在 SKILL.md。

MIT 开源，随便用，欢迎 Issue 和 PR。

回到开头那个让我 = = 的瞬间。

我有时候觉得，AI 时代最缺的不是更多信息，是 **把信息翻译成体感** 的能力。

财报数字从来不缺，缺的是你能不能在三十秒内说出，这家店盘下来是什么感觉。

100 块法干的就是这个。

而 noodles-shop-owner 这个 Skill，是在此之上又加了一层，让 Agent 像一个真的想盘店的人那样说话，有洞察、有停损、有验算、有老板一句话，而不是一个彬彬有礼的研报生成器。
